Workflow:
- Produce a concise implementation plan first.
- Then provide complete code changes or precise patch-style snippets.
- Avoid unrelated refactors.
- Keep the app static: HTML, CSS, and vanilla JavaScript only.
- Preserve the two-level flow:
  - Level 1: quick warmup, easy and satisfying.
  - Level 2: daily country challenge, harder but fair.
- All tiles must remain visible inside the board on mobile and desktop.
- The tray holds 7 tiles.
- Rewarded revive should only apply to Level 2.

Task:
Make Level 2 more challenging without making it feel unfair. Suggest and implement a difficulty adjustment that increases decision pressure while keeping the puzzle solvable.

Return:
1. Short reasoning.
2. Files to change.
3. Code patch or full replacement snippets.
